# Code Style Guide

This document defines the code style conventions for VeeCode DevPortal projects.

## Backstage Code Style

Follow the Backstage code style conventions as defined in:

- [Backstage Style Guide](https://github.com/backstage/backstage/blob/master/STYLE.md)

## TypeScript

### General Principles

- **Strict mode enabled** — All projects use `"strict": true`
- **Explicit types** — Prefer explicit types over inference for public APIs
- **No `any`** — Avoid `any`; use `unknown` when type is truly unknown
